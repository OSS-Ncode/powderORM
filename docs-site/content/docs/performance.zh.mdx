title: 性能
description: 为什么比原生驱动更快
---

Powder 不会逐行跨越 FFI 边界传递查询结果。Rust 核心将整个结果集编码为
**一个 PCB 列式缓冲区**，一次性传输，各语言的读取器再在该缓冲区之上
构建零拷贝视图。

<BenchmarkExplorer locale="zh" />

在全部五个语言标签页中，**都比该语言的原生 SQL 驱动更快。**所有运行器
均使用相同的数据、相同的 SQL 执行，并且必须通过行数 + 校验和的交叉验证。

对于重复执行的相同只读查询（SQLite `:memory:`），结果缓存可在
**0.02~0.03ms** 内返回——通过写入时全量失效、文件数据库的 `data_version`
监控、以及对非确定性函数的拦截这三重门禁来保证正确性。

## ORM 层的开销有多大？

几乎没有。SQL 骨架在 `powder generate` 阶段就已生成完毕（AOT），
运行时通过三种缓存消除了重复查询中的字符串处理开销：

- **行工厂缓存** —— 每张表生成一次的行构造函数（固定的 hidden class）
- **where 形状缓存** —— 将 WHERE 片段按谓词形态进行记忆化（值不参与缓存键）
- **ORDER BY 缓存** —— 排序片段采用相同方式记忆化
