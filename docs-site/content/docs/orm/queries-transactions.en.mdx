## Named queries (custom SQL)

Declare complex SQL in the schema's `queries` section. `powder generate`
compiles the `:name` placeholders into positional binds **at generation time**
and produces typed methods.

```json
"queries": {
  "topUsers": {
    "sql": "SELECT id, name, score, active FROM users WHERE active = :active AND score >= :minScore ORDER BY score DESC",
    "params": { "active": "bool", "minScore": "float" },
    "returns": "users"
  }
}
```

```ts
const rows = await db.$queries.topUsers({ active: true, minScore: 5 });
// rows: Users[]  (typed when `returns` is set)
```

```python
rows = await db.queries.top_users(active=True, min_score=5)
```

Validation happens at generation time: undeclared placeholders, unused params,
and a non-existent `returns` table are all rejected by `powder generate`. `:x`
inside string literals and `::` casts are left untouched.

## Transactions

Return to COMMIT, throw to ROLLBACK. Nested calls use SAVEPOINT, so an inner
failure doesn't erase the outer work.

```ts
await db.$transaction(async (tx) => {
  await tx.users.create({ id: 3, name: "carol", score: 7.5, active: true });
  await tx.posts.create({ id: 9, user_id: 3, title: "hi" });

  try {
    await tx.$transaction(async (inner) => {
      await inner.posts.create({ id: 10, user_id: 3, title: "oops" });
      throw new Error("cancel");      // ← only this insert rolls back
    });
  } catch {}
});
```

```python
async with db.transaction():
    await db.users.create({"id": 3, "name": "carol", "score": 7.5, "active": True})
```

The same semantics exist in Kotlin (`db.transaction { }`), Java, Go, C++, and C#.
