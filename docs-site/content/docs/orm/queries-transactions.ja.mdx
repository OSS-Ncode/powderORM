## 名前付きクエリ (カスタム SQL)

複雑な SQL はスキーマの `queries` セクションに宣言してください。
`powder generate` が `:名前` プレースホルダーを**生成時に**位置バインドへ
コンパイルし、typed メソッドを作成します。

```json
"queries": {
  "topUsers": {
    "sql": "SELECT id, name, score, active FROM users WHERE active = :active AND score >= :minScore ORDER BY score DESC",
    "params": { "active": "bool", "minScore": "float" },
    "returns": "users"
  }
}
```

```ts
const rows = await db.$queries.topUsers({ active: true, minScore: 5 });
// rows: Users[]  (returns が指定されていれば typed)
```

```python
rows = await db.queries.top_users(active=True, min_score=5)
```

検証は生成時点で完了します: 未宣言のプレースホルダー、未使用のパラメータ、
存在しない `returns` テーブルは `powder generate` が拒否します。文字列
リテラル内の `:x` や `::` キャストには手を加えません。

## トランザクション

正常終了で COMMIT、例外送出で ROLLBACK。ネストした呼び出しは SAVEPOINT
になるため、内側の失敗が外側の処理を消してしまうことはありません。

```ts
await db.$transaction(async (tx) => {
  await tx.users.create({ id: 3, name: "carol", score: 7.5, active: true });
  await tx.posts.create({ id: 9, user_id: 3, title: "hi" });

  try {
    await tx.$transaction(async (inner) => {
      await inner.posts.create({ id: 10, user_id: 3, title: "oops" });
      throw new Error("cancel");      // ← この insert だけがロールバックされる
    });
  } catch {}
});
```

```python
async with db.transaction():
    await db.users.create({"id": 3, "name": "carol", "score": 7.5, "active": True})
```

同じ意味論が Kotlin(`db.transaction { }`)、Java、Go、C++、C# にもあります。
