## 개념

Powder에서 외래 키(FK)가 곧 관계입니다. 별도로 관계를 선언하지 않아요.

```json
"posts": {
  "columns": {
    "id":      { "type": "int", "primaryKey": true },
    "user_id": { "type": "int", "references": { "table": "users", "column": "id" } },
    "title":   { "type": "text" }
  }
}
```

이 `references` 하나로 두 관계가 동시에 생깁니다:

- `posts.user` — belongsTo(자식 → 부모, 단건)
- `users.posts` — hasMany(부모 → 자식, 배열)

migrate 두 종류를 구분해두세요:

- `powder migrate --db app.db` — 가산적. 테이블/컬럼 추가만.
- `powder migrate --db app.db --rebuild` — 파괴적 변경(컬럼 삭제/타입 변경
  등)까지 반영. SQLite에서 데이터는 보존됩니다.

삭제 시 주의: Powder는 FK를 선언만 하고 `ON DELETE CASCADE`는 어떤
DB에도 만들지 않습니다. `delete()`도 그냥 `DELETE FROM`이라 앱 레벨
cascade도 없어요. 자식 행이 남은 부모를 지우면 PostgreSQL/MySQL은 즉시
에러를 내고, SQLite는 커넥션에서 `PRAGMA foreign_keys = ON`을 직접 켜두지
않으면 조용히 성공하면서 자식이 고아로 남습니다. (자세한 내용은
[관계와 include](/ko/docs/orm/relations) 참고.)

## 실습

`comments` 없이, `posts`만 있는 스키마에서 시작해서 관계를 실습해봅니다.

```ts
// 배치 로드 — 관계 레벨당 쿼리 1번 (N+1 없음)
const posts = await db.posts.findMany({ include: { user: true } });
posts[0].user?.name;

// 반대 방향 — 유저 기준으로 그 유저의 글 목록
const withPosts = await db.users.findMany({ include: { posts: true } });
withPosts[0].posts;   // 배열

// 중첩 include — 각 레벨이 배치 1쿼리
const deep = await db.posts.findMany({
  include: { user: { include: { posts: true } } },
});
```

`user_id`가 가리키는 유저를 지워보고 무슨 일이 일어나는지 직접
확인해보세요 (SQLite 기본값이면 조용히 성공하고 `posts.user_id`가 존재하지
않는 유저를 가리키게 됩니다 — 위 "개념"에서 설명한 그대로입니다).

## 문제

`comments` 테이블을 추가하세요. 요구사항:

- 댓글은 게시글(`posts`) 하나에 달린다
- 댓글은 작성자(`users`) 하나가 쓴다 (즉 `posts`와 `users` 둘 다 참조)
- 댓글 내용은 `body`(text)

스키마를 작성하고, 특정 게시글의 댓글들을 작성자 정보와 함께 N+1 없이
가져오는 쿼리까지 짜보세요.

<details>
<summary>힌트</summary>

- FK 컬럼은 관례상 `<참조테이블_단수>_id` 이름을 씁니다 (`post_id`,
  `user_id`). 컬럼 이름 자체는 자유지만 이 관례를 따르면 생성되는 관계
  이름(`comment.post`, `comment.user`)도 자연스러워집니다.
- 한 테이블이 FK를 두 개 가지는 것 — `posts`도 참조하고 `users`도
  참조하는 것 — 은 전혀 특별하지 않습니다. 컬럼마다 `references`를 따로
  달면 됩니다.
- 댓글 목록 조회는 `db.comments.findMany({ where: { post_id: ... }, include: { user: true } })`
  형태가 됩니다.

</details>

<details>
<summary>정답</summary>

```json
"comments": {
  "columns": {
    "id":      { "type": "int", "primaryKey": true },
    "post_id": { "type": "int", "references": { "table": "posts", "column": "id" } },
    "user_id": { "type": "int", "references": { "table": "users", "column": "id" } },
    "body":    { "type": "text" }
  }
}
```

```ts
const comments = await db.comments.findMany({
  where: { post_id: 1 },
  include: { user: true },
});
comments[0].user?.name;   // N+1 없이 작성자 이름까지 한 번에
```

이제 `posts.comments`(hasMany), `users.comments`(hasMany)도 자동으로
생겼으니, 반대 방향으로도 조회할 수 있습니다:

```ts
const postWithComments = await db.posts.findMany({
  include: { comments: { include: { user: true } } },
});
```

</details>
