## 문제 1 — 좋아요 기능

`posts`에 "좋아요"를 추가하려 합니다. 한 유저는 같은 게시글에 좋아요를
**한 번만** 누를 수 있어야 합니다.

<details>
<summary>힌트</summary>

다대다 관계는 Powder에 별도 문법이 없습니다 — **중간 테이블**을 직접
만들어서 두 개의 belongsTo로 표현하세요. "한 번만"이라는 제약은 스키마
레벨에서 강제할 방법이 마땅치 않으니(복합 유니크 키는 이 스키마 문법에
없음), 애플리케이션 코드에서 먼저 `where`로 존재 여부를 확인하고
insert하는 방식을 생각해보세요.

</details>

<details>
<summary>정답</summary>

```json
"likes": {
  "columns": {
    "id":      { "type": "int", "primaryKey": true },
    "post_id": { "type": "int", "references": { "table": "posts", "column": "id" } },
    "user_id": { "type": "int", "references": { "table": "users", "column": "id" } }
  }
}
```

```ts
async function likePost(userId: number, postId: number) {
  const already = await db.likes
    .where("user_id", "=", userId)
    .where("post_id", "=", postId)
    .exists();
  if (already) return;
  await db.likes.create({ id: /* 다음 id 채번 로직 */ 1, user_id: userId, post_id: postId });
}

// 게시글의 좋아요 수
const count = await db.likes.where("post_id", "=", 1).count();
```

</details>

## 문제 2 — 탈퇴한 유저 처리

유저를 삭제하면 그 유저가 쓴 게시글은 어떻게 될까요? 2장에서 배운 내용을
바탕으로, `users` 삭제 시 실제로 무슨 일이 일어나는지 SQLite와
PostgreSQL 각각에서 설명하고, "탈퇴해도 글은 남기고 싶다"는 요구사항을
어떻게 구현할지 제안해보세요.

<details>
<summary>힌트</summary>

2장에서 다룬 "삭제 시 주의" 내용을 다시 보세요. Powder는 cascade를 만들지
않습니다. "글은 남기고 싶다"는 건 애초에 하드 delete가 아니라 다른
방식이 필요하다는 뜻입니다.

</details>

<details>
<summary>정답</summary>

- **SQLite**: 커넥션에서 `PRAGMA foreign_keys = ON`을 켜지 않았다면
  `users` 삭제가 조용히 성공하고, 남은 `posts.user_id`는 존재하지 않는
  유저를 가리키는 고아 데이터가 됩니다. 켰다면 FK 위반으로 즉시 에러가
  납니다.
- **PostgreSQL**: 기본적으로 FK가 강제되므로, 그 유저의 글이 하나라도
  남아있으면 삭제 자체가 에러로 실패합니다.
- **"글은 남기고 싶다"**는 요구사항이면애초에 물리 삭제(`delete()`)를
  쓰지 말고, `users`에 `deleted_at`(nullable datetime) 같은 컬럼을 추가해
  "삭제"를 `update({ where: { id }, data: { deleted_at: now } })`로
  구현하는 소프트 삭제 패턴을 씁니다. 조회할 때는
  `where("deleted_at", "=", null)`로 걸러주면 됩니다.

</details>

## 문제 3 — 인기 게시글 랭킹

각 게시글의 좋아요 수를 계산해서, 좋아요가 많은 순으로 상위 5개 게시글과
작성자 이름을 가져오는 코드를 짜보세요. (문제 1의 `likes` 테이블 기준)

<details>
<summary>힌트</summary>

Powder의 `findMany`/`include`는 "관계 개수로 정렬"을 직접 지원하지
않습니다. 좋아요 수처럼 집계가 필요한 정렬은 애플리케이션에서 계산하거나,
[명명 쿼리](/docs/orm/queries-transactions)로 raw SQL(`GROUP BY` +
`COUNT`)을 쓰는 게 정확합니다.

</details>

<details>
<summary>정답</summary>

애플리케이션 레벨 버전(게시글이 많지 않을 때 충분히 실용적):

```ts
const posts = await db.posts.findMany({ include: { user: true, likes: true } });
const ranked = posts
  .map((p) => ({ ...p, likeCount: p.likes.length }))
  .sort((a, b) => b.likeCount - a.likeCount)
  .slice(0, 5);

ranked.forEach((p) => console.log(p.user?.name, p.title, p.likeCount));
```

게시글이 많아지면 DB에서 집계하는 게 맞습니다 — `powder.schema.json`의
`queries` 섹션에 명명 쿼리로 `GROUP BY post_id ORDER BY COUNT(*) DESC
LIMIT 5` 형태의 SQL을 등록하면, 생성 시점에 컴파일되어 typed 메서드로
쓸 수 있습니다. 자세한 문법은 [명명 쿼리와 트랜잭션](/docs/orm/queries-transactions)
문서를 참고하세요.

</details>
