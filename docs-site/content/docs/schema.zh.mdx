单一的 schema 文件即可派生出所有内容：DDL、迁移（migration）、校验、
typed 模型、关系（relation）与命名查询。

```json
{
  "$schema": "./powder.schema.schema.json",
  "tables": {
    "users": {
      "columns": {
        "id":     { "type": "int",   "primaryKey": true },
        "name":   { "type": "text" },
        "score":  { "type": "float", "nullable": true },
        "active": { "type": "bool" }
      }
    },
    "posts": {
      "columns": {
        "id":      { "type": "int", "primaryKey": true },
        "user_id": { "type": "int", "references": { "table": "users", "column": "id" } },
        "title":   { "type": "text" }
      }
    }
  }
}
```

## 类型

`int` · `float` · `text` · `bool` —— 与 PCB 线上格式以零拷贝方式传输的
类型集合完全一致。列默认是 NOT NULL，可通过 `"nullable": true` 放开限制。

## 键

- **主键**：`"primaryKey": true`。作用于多个列即构成复合主键
  （`PRIMARY KEY (a, b)`），`validate` 还会校验列的顺序。
- **外键**：单列外键通过该列的 `references` 指定；复合外键通过表级的
  `foreignKeys` 指定。解析时会检查目标是否存在、类型是否一致，并自动
  派生出 ORM 关系（`posts.user` / `users.posts`）。

```json
"foreignKeys": [
  { "columns": ["order_id", "order_year"],
    "references": { "table": "orders", "columns": ["id", "year"] } }
]
```

## 编辑器自动补全

`powder init` / `powder new` 还会生成 `powder.schema.schema.json`
（一份 JSON Schema），文件首行的 `$schema` 会指向它。这样 VS Code
即可自动补全键、类型与查询约定，并在输入时即时标记无效值。

参见[命名查询与事务](/zh/docs/orm/queries-transactions)——`queries`
