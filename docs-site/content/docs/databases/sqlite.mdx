SQLite는 Powder의 기본 백엔드예요. 설치할 서버가 없고, 파일 하나 또는
인메모리로 바로 시작합니다. 마이그레이션과 결과 캐시까지 전부 지원해요.

## 연결

```
sqlite::memory:     인메모리 (프로세스 수명)
sqlite://./app.db   파일 (명시적)
./app.db            파일 (경로만)
```

## 지원 범위

| 기능 | 지원 |
|---|---|
| 런타임 쿼리 | ✅ |
| DDL 생성 (`powder ddl --dialect sqlite`) | ✅ |
| migrate / validate / seed | ✅ (`--rebuild` 포함) |
| 결과 캐시 | ✅ |

## 결과 캐시 (SQLite 전용)

반복되는 동일 읽기 쿼리는 결과 캐시에서 **0.02~0.03ms**에 답해요. 정확성은
3중 게이트로 지켜집니다:

- 쓰기가 일어나면 전체 무효화
- 파일 DB는 `data_version`을 감시해 외부 변경 감지
- 비결정 함수(예: `random()`)가 든 쿼리는 캐시 대상에서 제외

## 파괴적 마이그레이션 — `--rebuild`

SQLite는 컬럼 삭제·타입 변경을 직접 못 해요. `--rebuild`가 공식 재구축
패턴으로 이를 해소하면서 **살아남는 컬럼의 데이터는 보존**합니다.

```bash
powder migrate --db app.db --rebuild
```

타입 매핑: `int → INTEGER`, `float → REAL`, `text → TEXT`, `bool → INTEGER`.
