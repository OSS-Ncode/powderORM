SQLiteはPowderのデフォルトバックエンドです。インストールするサーバーは不要で、ファイル一つ、
またはインメモリですぐに始められます。マイグレーションと結果キャッシュまで全てサポートしています。

## 接続

```
sqlite::memory:     インメモリ (プロセスの寿命)
sqlite://./app.db   ファイル (明示的)
./app.db            ファイル (パスのみ)
```

## サポート範囲

| 機能 | サポート |
|---|---|
| ランタイムクエリ | ✅ |
| DDL生成 (`powder ddl --dialect sqlite`) | ✅ |
| migrate / validate / seed | ✅ (`--rebuild` を含む) |
| 結果キャッシュ | ✅ |

## 結果キャッシュ (SQLite専用)

繰り返される同一の読み取りクエリは、結果キャッシュから**0.02〜0.03ms**で応答します。
正確性は3重のゲートで守られます。

- 書き込みが発生すると全体を無効化
- ファイルDBは `data_version` を監視して外部の変更を検知
- 非決定的関数(例: `random()`)を含むクエリはキャッシュ対象から除外

## 破壊的マイグレーション — `--rebuild`

SQLiteはカラムの削除・型変更を直接行えません。`--rebuild` が公式の再構築
パターンでこれを解決しつつ、**生き残るカラムのデータは保持**します。

```bash
powder migrate --db app.db --rebuild
```

型マッピング: `int → INTEGER`, `float → REAL`, `text → TEXT`, `bool → INTEGER`。
