`crates/powder-ffi`가 안정 C ABI를 내보내요. Go·C#·C++ 바인딩이 전부 이
표면 위에 얹혀 있습니다. 헤더 `powder.h` 하나면 됩니다.

## 사용법

```c
#include "powder.h"

PowderClient *db = powder_connect("sqlite::memory:");
powder_execute(db, "CREATE TABLE t (id INTEGER, name TEXT)", NULL);
powder_execute(db, "INSERT INTO t VALUES (?, ?)", "[1, \"alice\"]");

size_t len;
unsigned char *pcb = powder_query(db, "SELECT * FROM t", NULL, &len);
/* pcb는 PCB 컬럼 버퍼 — 포맷은 docs/FORMAT.md, 디코더 예시는 C++ 래퍼 참고 */
powder_free_buffer(pcb, len);
powder_close(db);
```

## 규칙

- **파라미터는 JSON 배열 문자열**로 전달해요: `"[1, \"a\", true, null]"`.
  C 표면을 단순 포인터·정수로 유지하기 위한 선택이에요.
- **오류는 NULL / -1** + `powder_last_error()`(스레드 로컬, borrowed) 또는
  `powder_last_error_copy()`로 확인합니다.
- `powder_query`가 준 버퍼는 **반환된 길이 그대로** `powder_free_buffer`로
  해제하세요.

## ORM

C에서도 Powder ORM을 완전 지원해요 — 공유 Rust 엔진(`powder_core::orm`)의
JSON op 스펙을 그대로 씁니다. `powder.schema.json` 텍스트로 스키마 핸들을
한 번 만들고, 연산마다 JSON 객체 하나를 넘겨요. 연산·문법은 모든 언어와
동일: `findMany`/`findFirst`/`create`/`createMany`/`update`/`delete`/
`deleteAll`/`count`/`exists`/`aggregate`/`groupBy`(+having), 중첩
`AND`/`OR`/`NOT` where(`eq ne gt gte lt lte like in`), `include`/`join`.

```c
PowderOrmSchema *schema = powder_orm_schema_new(schema_json);

powder_orm_execute(db, schema,
    "{\"op\":\"create\",\"table\":\"users\","
    "\"data\":{\"id\":1,\"name\":\"alice\",\"score\":9.5,\"active\":true}}");

size_t len = 0;
unsigned char *rows = powder_orm_find_json(db, schema,
    "{\"op\":\"findMany\",\"table\":\"users\","
    "\"where\":{\"active\":true,\"score\":{\"gte\":5}},"
    "\"orderBy\":{\"score\":\"desc\"},\"limit\":10}", &len);
/* rows = UTF-8 JSON 배열 (len 바이트) */
powder_free_buffer(rows, len);
powder_orm_schema_free(schema);
```

변경/카운트 op는 `powder_orm_execute`(영향 행 수, 실패 -1), 행을 돌려주는
op는 `powder_orm_find_json`(JSON, `powder_free_buffer`로 해제)입니다.

## 빌드 (Windows / MSVC)

```bat
cargo build -p powder-ffi --release
cl /W3 /utf-8 test\test_powder.c /I include /link <target>\release\powder_ffi.dll.lib
```

Unix: `cc test/test_powder.c -Iinclude -lpowder_ffi -L<target>/release`.

PCB를 직접 디코드하기보다 편한 래퍼를 원하면 [C++](/ko/docs/bindings/cpp)
헤더가 RAII + 디코더를 제공해요.
